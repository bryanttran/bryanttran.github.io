SlugSearch

What I Did

As a lead UI/UX developer, I helped create a simple and intuitive user interface, while helping parse data to put into the application.

What we Utilized

HTML5
CSS3
AngularJS
Ionic Framework
Scrum Methodology


Parsed UCSC's class search page for information onto a mobile application.

SlugSearch is a mobile application that lets you search through UC Santa Cruz's past and current classes. The aim was to have an application on mobile to search for classes efficiently, since UC Santa Cruz's website is not mobile friendly, and has many bugs compared to its desktop application. Our application also includes ratings scraped from the website RateMyProfessor which includes an average ratings from the reviews and average grade on the professor. explain a little more, rate my professor

Development steps


1. Research

One of the main issues in the research stage was to learn a whole new framework and language. We decided to use the Ionic Framework, since it is capable of building on both iOS and Android. By using AngularJS' GET and POST requests, we were able to effectively scrape the UC Santa Cruz's class search page.

We had scrum meetings three times a week. We received guidance from the teacher assistant who advised us with our development and scrum terminology. Using a burn up chart, story plots, three sprints, and a backlog, we succesfully developed an application that fulfilled our main functions within a span of 8 weeks.

2. Development

The first sprint consisted of scraping big data from the UC Santa Cruz search page, creating a basic UI design, and parsing the data into our application. Since we were still getting used to the Ionic Framework, we used a template to get us started for our user interface.

In the second sprint, we decided to start fresh in regards to front end development. We applied a more simplistic CSS to the application instead of using a template in order to make the application more aesthetically pleasing. The team also succesfully scraped RateMyProfessor, so that the content would be available in the application. Additionally, we were able to scrape the description for each class, so that the user can be presented with more information.

The last sprint mainly consisted of quality assurance, and touching up our UI/UX design. We planned reusable tests to confirm the functionality of the application was stable. We were able to deploy the application on a tablet as well as create a shopping cart tab in order to have a cart to store desired classes.

3. Launch

At the end of the quarter, we presented our project, what we did during our three sprints, and a live quality assurance test to confirm the stability of the application to the class. Some of these story plots consisted of push notifications and a live schedule to see time slots of the week. edit this more

UCSC Slugbook Exchange

What I Did

Created a web application to allow UC Santa Cruz students to buy and sell books, similar to a forum.

What I Utilized

HTML5
CSS3
Web2Py Framework


Created a web application to effectively buy and sell books to peers.

UCSC Slugbook Exchange is a web application to trade in books of all subjects. The application was used as a web application class project. The application was created on the Web2Py Framework which allowed us to manipulate SQLite databases, built in forms, and premade classes from the BootStrap CSS.

Development steps


1. Research

My main goal of this project was to allow all students pertaining to UC Santa Cruz to efficiently buy and sell books to other students rather than sell back to the school at a ridiculous price. While learning the ins and outs of the Web2Py Framework, my original project was not the same as the final outcome.

Using the built in resources of the Web2Py Framework, I had planned certain content such as viewing as a guest, viewing as a user, creation of posts, preventing mainpulation of the url, and secure login authentication.

2. Development

The first thing I did when creating this application was map out the pages of what I need with the basic working functionalities. A standard board html page, a posts html page, and a functional post button to show the board without refreshing the page which all send data to and from each page. Utilizing the Web2Py Framework allowed for smooth creation of databases, login requirements to post, and creation and deletion of posts. After basic functionality was finished, I ventured onto user experience content, which some were implemented, and some were not.

3. Launch

With this being a class project, at the end of the quarter we gave a presentation of our project. Some of the more time consuming features were not implemented due to the time constraints. Some features that were not implemented were FaceBook login, a dedicated hosting server, ucsc.edu email check, and direct messaging.
